### 파일명 및 커밋 컨벤션

- 기본 형식: `플랫폼_문제번호_문제명`
  책에만 있는 문제는 `문제 번호_문제명` 형식으로 작성합니다.

- 디렉토리 및 파일명 규칙: 공백은 언더바(`_`)로 치환하여 작성

- 커밋 메시지 규칙: 대괄호 `[ ]` 안에 플랫폼과 문제 식별자를 명시합니다.

[예시 1] 프로그래머스 등 플랫폼 문제

- 디렉토리명: `PGS_42576_완주하지_못한_선수`

- 파일명: `완주하지_못한_선수.js`

- 커밋: `[PGS_42576] 완주하지 못한 선수`

[예시 2] 책에만 존재하는 문제

- 디렉토리명: `01_배열_정렬하기`

- 파일명: `배열_정렬하기.js`

- 커밋: `[01]  배열 정렬하기`

## 🚀 코드 실행 방법

로컬에 Node.js가 설치되어 있다면 별도의 패키지 설치 없이 터미널에서 즉시 코드를 실행하여 결과를 확인할 수 있습니다.

```bash
# 방법 1. 최상위 경로에서 상대 경로로 바로 실행
node yuwol/week1/PGS_42576_완주하지_못한_선수/완주하지_못한_선수.js

# 방법 2. 해당 문제 디렉토리로 이동 후 실행
cd yuwol/week1/PGS_42576_완주하지_못한_선수
node 완주하지_못한_선수.js
```

## 🔄 작업 워크플로우 (Fork & PR)

1. **Clone**: 스터디 공통 레포지토리를 로컬 환경으로 가져옵니다.

   ```bash
   git clone https://github.com/woowacourse-study/2026-fe-algorithm-study.git
   ```

2. **Branch 이동**: 본인의 영문 이름으로 생성된 브랜치로 이동합니다.

   ```bash
   git checkout yuwol
   ```

3. **작업 진행**: 본인 디렉토리 내 해당 주차 폴더(e.g. `yuwol/week1/..`)에 컨벤션에 맞춰 풀이 코드를 작성합니다.

   (❗️주의: 병합 충돌을 방지하기 위해 **본인의 디렉토리 외부 파일은 절대 수정하지 않습니다**)

4. **Commit & Push**: 알고리즘 풀이가 완료되면 커밋 컨벤션에 맞춰 커밋 후 본인의 브랜치에 Push 합니다.

5. **Pull Request**: 본인의 브랜치에서 `main` 브랜치로 PR을 생성합니다.

6. **Code Review & Merge**: 각 스터디원들의 PR이 등록되면 다 함께 코드 리뷰를 진행하고, 아래의 승인 요건이 충족되면 `main`에 merge 합니다.

✅ **PR 및 코드 리뷰 규칙**

- PR 제출: 매주 월요일 18:00까지 본인이 해결한 문제에 대한 풀이를 PR로 제출합니다.
- 코드 리뷰: 스터디원 누구나 모든 PR에 자유롭게 의견과 리뷰를 남길 수 있습니다.
- 승인 권한: PR에 대한 approve는 모든 스터디원이 가능합니다.
- merge 조건: 본인을 제외한 모든 스터디원의 approve가 된 PR에 한해서만 main 브랜치로 merge가 가능합니다.
