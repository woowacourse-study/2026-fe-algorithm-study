## 💡 어떤 문제를 풀었나요?
- 04_모의고사
## 🔑 풀이 접근 방식
* 시간복잡도: O(N) 
1. 수포자의 찍는 방식 패턴을 배열로 저장
2. 정답과 패턴속 값이 일치하는 수를 저장
3. 저장한 배열에서의 최댓값을 구해서 최댓값과 같은 것만 결과 배열에 push

## ❓ 고민한 점 & 리뷰 요청 사항
패턴에서 결과값과 비교하는 로직을 구현하는 과정이 오래걸렸다.

## 💡 어떤 문제를 풀었나요?
- 05_행렬의_곱셈
## 🔑 풀이 접근 방식
* 시간복잡도: O(N^3)
1. 결과를 담을 2차원 배열을 초기화한다.
2. arr1의 행 수 만큼 반복하면서 초기화한 결과 배열에 행렬곱 값을 push

## ❓ 고민한 점 & 리뷰 요청 사항
행렬곱 계산을 까먹어서 고민했다.

## 💡 어떤 문제를 풀었나요?
- 06_실패율
## 🔑 풀이 접근 방식
* 시간복잡도: O(N^2)
1. 각 스테이지마다 실패한 사람 수 저장
2. 실패율 계산하고 전체 플레이어수에서 해당 스테이지에서 실패한 플레이어 수를 제거
3. 정렬해서 answer배열에 푸시

## ❓ 고민한 점 & 리뷰 요청 사항
1. 객체로 저장할지, 그냥 배열로 저장할지 고민하다가
생각하기 편하게 객체로 셀렉함.

2. 스테이지별 실패한 사람 수를 미리 저장해두기 위해 filter를 사용했다. 
이것 때문에 시간복잡도 손해를 보는 것 같아서 AI에게 물어봤더니, 
반복문으로 stages를 한 번만 순회하게 해서 시간복잡도를 감소시킴.
O(N^2)  =>  O(Nlog(N))

## 💡 어떤 문제를 풀었나요?
- 07_방문_길이
## 🔑 풀이 접근 방식
1. 좌표 초기화 (x = 0 , y = 0)
2. 누적 좌표를 반복문 내에서 정의
3. UDRL에 따라 누적좌표값 할당
4. 같은 경로 중복 제거를 위해 항상 작은 좌표 -> 큰 좌표 순으로 저장
5. Set으로 중복 제거 
-

## ❓ 고민한 점 & 리뷰 요청 사항
좌표 이동 경로가 다르더라도 결국 같은 경로 이동일 경우
(ex. (3,0) -> (3,3) 과 (3,3) -> (3,0)) 
이 상황 구분을 어떻게 해야할지 고민을 많이 했다.

항상 작은 좌표가 앞에 오도록 조건을 걸고, 중복 제거를 하는 방향으로 해결했다.
